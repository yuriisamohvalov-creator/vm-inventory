# Руководство пользователя VM Inventory

## 1. Назначение системы
VM Inventory — веб-приложение для учета виртуальных машин (ВМ), их привязки к структуре компании (Департамент → Стрим → ИС), управления пулами и выгрузки отчетов.

## 2. Вход в систему
1. Откройте главную страницу приложения.
2. Введите логин и пароль.
3. Нажмите **Войти**.

> Если администратор включил флаг обязательной смены пароля, выполните смену пароля при первом входе.

## 3. Раздел «Виртуальные машины»
### Что можно делать
- Просматривать список ВМ.
- Добавлять ВМ (для роли с правами записи).
- Редактировать параметры ВМ.
- Удалять/восстанавливать ВМ.

### Ключевые поля ВМ
- **FQDN** — уникальное имя ВМ.
- **IP** — IP-адрес; при дублировании система выдаст предупреждение.
- **CPU / RAM / Disk** — ресурсы.
- **Instance** — номер инстанса (используется в логике пулов).
- **ИС** — привязка к информационной системе.
- **Теги** — ОС + код ИС + пользовательские теги.

## 4. Раздел «Общие пулы»
### Что делает раздел
- Позволяет объединять ВМ в пулы для совместного управления.

### Ограничения
- ВМ в одном пуле должны соответствовать правилам системы (в частности, по `instance`).

## 5. Раздел «Отчеты»
### Возможности
- Просмотр иерархии: Департамент → Стрим → ИС → ВМ.
- Экспорт отчетов в форматы:
  - PDF
  - XLSX
  - JSON

## 6. Раздел «Администрирование» (для администраторов)
- Управление департаментами, стримами, ИС.
- Массовый импорт данных из JSON.
- Управление пользователями и ролями.

## 7. Раздел «Инструкции»
В левой панели доступна ссылка **Инструкции**.
- Раздел открывается в новой вкладке браузера.
- Включает:
  - Руководство пользователя.
  - Руководство администратора по развертыванию и созданию пользователей.

## 8. Типовые сценарии
### Добавление новой ВМ
1. Откройте **Виртуальные машины**.
2. Нажмите **Добавить ВМ**.
3. Заполните обязательные поля.
4. Нажмите **Сохранить**.

### Выгрузка отчета
1. Откройте **Отчеты**.
2. Выберите формат экспорта (PDF/XLSX/JSON).
3. Сохраните файл локально.

## 9. Частые ошибки и решения
- **Неверный формат IP**: используйте формат `xxx.xxx.xxx.xxx`.
- **Недостаточно прав**: обратитесь к администратору для назначения роли.
- **Ошибка авторизации**: выйдите и войдите повторно.
